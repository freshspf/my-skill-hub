# handoff

会话中断前的现场保存工具。生成的 HANDOFF.md 让下一个 agent 直接接上当前状态，不用重新探索。

## 核心原则

**失败路径比成功路径更有价值**：试过但走不通的方案和原因必须优先提取，否则下一个 agent 大概率会走同样的弯路。

## 执行步骤

### Step 1 — 确定存放位置

HANDOFF.md 写在 **当前工作目录的根目录**。

如果当前会话在探索某个子模块（如 `pgyagent/`），HANDOFF.md 写在该子目录下（如 `pgyagent/HANDOFF.md`）。

### Step 2 — 从会话上下文提取热状态

按优先级提取以下信息：

1. **❌ 试过但行不通的路径**（最高优先级）
   - 尝试过什么方案、命令、文件路径
   - 为什么行不通（报错、逻辑错误、不适用）

2. **中断时在做什么**
   - 正在探索/实现哪个功能
   - 具体到哪个类、方法、行号

3. **本会话的关键发现**（未写入正式文档/commit 的）
   - 探索得到的结论
   - 发现的设计决策或隐含约束

4. **正在验证的假设**
   - 还没从代码/测试找到证据的猜测

5. **下一步**（具体到文件/方法）
   - 不是泛泛的"继续探索"，而是具体的下一个动作

6. **已修改的文件**
   - 本会话写了哪些文档/代码，改了什么

### Step 3 — 生成 HANDOFF.md

按固定模板输出。**空节点不省略，写"暂无"**，让下一个 agent 知道这项已被考虑过。

```markdown
# HANDOFF - [任务/模块名] - YYYY-MM-DD Session N

> **用途**: 跨会话热状态接力。下次会话开始时先读本文件，完成后删除。
> **适用项目**: [项目名/目录]

## 中断时在做什么
（一句话：正在探索/实现哪个功能，卡在哪一步）

## 本会话的关键发现（未写入正式文档）
（探索或实现过程中得到但还没整理的结论）

## ❌ 试过但行不通的路径
（最有价值——下一个 agent 不会重走弯路）
- ❌ [方案/文件/命令]: [失败原因]

## 正在验证的假设
（还没从代码/测试找到证据的猜测）
- [ ] [假设内容]

## 下一步（具体到文件/方法/行）
（比任务清单更细：具体类路径、行号、待读的方法）

## 本会话已修改的文件
- [文件路径]: [改了什么]
```

### Step 4 — 提示用户

生成完成后告知：
- HANDOFF.md 已写入哪个路径
- 如果是**正常结束**（非中断）：建议同时更新 progress.md（如有），然后删除 HANDOFF.md
- 如果是**中断退出**：直接退出，下次会话开始时先读 HANDOFF.md

---

## 各项目微调方式

skill 默认行为适用于所有项目。如需微调，在项目的 CLAUDE.md 里添加说明：

| 项目类型 | CLAUDE.md 里的微调 |
|---------|-------------------|
| 知识库探索 | "先读 HANDOFF.md，再读 progress.md" |
| 代码项目 | "HANDOFF.md 里必须包含 git diff 摘要" |
| 调试类任务 | "HANDOFF.md 里必须记录复现步骤" |

---

## HANDOFF.md 的生命周期

```plaintext
会话中断
  ↓
/handoff 生成 HANDOFF.md
  ↓
下次会话：先读 HANDOFF.md → 接上状态
  ↓
任务完成 → 更新 progress.md（如有）→ 删除 HANDOFF.md
```
