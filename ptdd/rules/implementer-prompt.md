# Implementer Subagent Prompt 模板

dispatch `general` subagent 执行单个任务时使用此模板。

**重要**：将任务完整文本直接粘贴进 prompt，不要让 subagent 自己去读计划文件。

```
你正在实现以下任务：

## 任务描述

[将计划文件中该任务的完整文本粘贴到这里，包含目标、文件、测试代码、实现代码、步骤]

## 场景上下文

[说明该任务在整体功能中的位置，已完成的前置任务是什么，有哪些依赖]

## 测试环境

[粘贴 agent_docs/rules/tdd_setup.md 的内容]

## 开始前

如果你对以下内容有疑问，**现在就提问**，不要在实现中途才问：
- 任务要求或验收标准
- 实现方式或技术细节
- 依赖关系或假设前提

## 你的职责

1. 严格按照任务步骤执行（RED → GREEN → REFACTOR）
2. 先写测试，让测试失败，再写实现，让测试通过
3. 不要实现任务范围之外的功能（YAGNI）
4. 完成后自查
5. **必须提交**：每个任务完成后必须执行 `git add` + `git commit`，这是强制要求，不是可选项

## 自查清单（提交前必须确认）

- [ ] 完整实现了任务描述中的所有要求
- [ ] 没有实现任务之外的功能
- [ ] 所有测试通过
- [ ] 覆盖率 ≥ 80%
- [ ] 已执行 git commit（commit message 格式：`feat/fix/refactor: [描述]`）

## 遇到困难时

以下情况请立即停止并上报，不要硬撑：
- 任务涉及的代码超出提供的上下文范围
- 测试持续无法通过且原因不明
- 需要修改计划范围之外的文件

## 返回格式

完成后报告：

**状态**：DONE / DONE_WITH_CONCERNS / BLOCKED / NEEDS_CONTEXT

**完成内容**：[实现了什么]

**测试结果**：[X 个测试通过，覆盖率 XX%]

**修改文件**：[文件列表]

**Commit**：[commit hash 和 message]

**问题或顾虑**（如有）：[描述]
```

**状态说明**：
- `DONE`：正常完成
- `DONE_WITH_CONCERNS`：完成但有疑虑，主线程需要关注
- `BLOCKED`：无法继续，需要主线程介入
- `NEEDS_CONTEXT`：缺少上下文，需要补充后重新 dispatch
