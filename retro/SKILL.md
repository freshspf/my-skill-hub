# 需求复盘文档生成 Skill

将当前会话中完成的需求复盘为一篇结构化文档，自动写入个人 Hi 文档空间根目录。

## 执行流程

### Step 1：读取用户画像

读取 `~/Gitroom/项目理解/个人画像.md`，获取用户身份、技术栈、工作风格等信息，确保"学习到的点"章节贴合用户水平。

### Step 2：获取 Git 上下文

```bash
git branch --show-current
git log --oneline -10
```

获取当前分支名和近期提交，用于填充"相关信息"章节和理解改动脉络。

### Step 3：组织复盘内容

从当前会话上下文中提取信息，按以下**四个固定章节**组织 Markdown 内容。所有内容必须来自当前会话上下文，不要编造会话中没有出现的信息。如果某个章节信息不足，精简但不跳过。

---

#### 章节一：需求背景

用 1-2 段文字简要说明：这个需求/问题是什么、为什么需要做、影响是什么。

如果有工单号、风险等级等结构化元数据，用一个信息表放在最前面，再跟文字描述。

#### 章节二：我做了什么

用第一人称描述具体做了什么。如果经历了多个迭代版本，按版本分小节（### 第一版 / ### 第二版 ...），每个版本说明：

- **核心思路**：一句话概括这个版本的改法
- **改动文件**：列出改了哪些文件、每个文件改了什么（无序列表）
- 如果有配置项、接口变更等，用表格呈现

如果只有一个版本，不需要分版本小节，直接描述即可。

如果有量化的改进效果（性能提升、风险消除等），在末尾用**对比表格**展示（优化前 vs 优化后）。

#### 章节三：学习到的点

**最重要的章节。** 提炼 3-12 个学习点，每个学习点格式：

```
### N. 一句话标题（可迁移的工程原则）

2-4 句展开说明，包含：
- 具体场景：在这次需求中遇到了什么情况
- 为什么重要：这个原则/教训的价值
- 行动指南：下次遇到类似场景怎么做

如果有代码对比（正确 vs 错误写法），用代码块展示。
用 **教训**、**关键认知**、**原则** 等加粗词引出核心 takeaway。
```

写作原则：

- 第一人称，口语化但不随意（"一开始我想..."、"被指出后才意识到..."）
- 每个学习点是**具体的行动指南**，不是空泛的大道理
  - ❌ "要注意代码安全"
  - ✅ "鉴权代码必须 deny by default——查不到关系等于无权限，不是跳过校验"
- 优先提炼：安全编码原则 > 代码审计技巧 > 架构设计模式 > 研发流程认知 > 工具使用技巧
- 如果有导师/同事的反馈导致改进，如实记录（"导师建议..."、"CR 时被指出..."）
- 学习点数量根据需求复杂度灵活调整，简单需求 3-5 个，复杂需求 8-12 个

#### 章节四：相关信息

用无序列表列出关键元数据：

- 最终分支名
- 历史分支（如有多个迭代）
- 改动文件列表
- 上下游服务
- 关联工单/PR 号
- 使用的框架/工具

---

### Step 4：处理 REDoc 特殊字符

执行 `hi docs:markdown-syntax` 查看转义规则。

正文中（代码块外）的特殊字符必须转义：

| 原始字符 | 转义写法 |
|---------|---------|
| `<` | `&lt;` |
| `>` | `&gt;` |
| `{` | `&#123;` |
| `}` | `&#125;` |
| `\|`（表格内非分隔符的竖线） | `&#124;` |

**代码块（\`\`\` 包裹）内不需要转义。**

### Step 5：生成文档

1. 将 Markdown 内容写入临时文件 `/tmp/retro_doc.md`（注意：content 不包含标题行，标题通过 --title 参数传）
2. 生成 operateCode：
   ```bash
   hi utils:generate-operate-code
   ```
3. 创建文档（不传 space-id 和 parent-id，默认个人空间根目录）：
   ```bash
   cat /tmp/retro_doc.md | hi docs:create --title "【复盘】<简要描述>（<标识符>）" --content - --operate-code "<operateCode>"
   ```

### Step 6：返回结果

向用户展示：
- 文档链接
- 提炼的学习点数量和概要

## 标题格式

`【复盘】<简要描述>（<标识符>）`

标识符优先级：工单号 > PR 号 > 分支名

示例：
- `【复盘】MCN代设价格授权绕过漏洞修复（VULN-260421-0004）`
- `【复盘】超级玩家投后数据-下载笔记list导出加速优化（feature/shenpengfei-download-opt）`

## 注意事项

- 复盘内容**全部来自当前会话上下文**，不编造
- content 通过管道 `cat file | hi docs:create --content -` 传入，避免 shell 转义问题
- 不要在 `--content` 中重复 `--title` 的内容
- 章节编号用中文数字（一、二、三、四）
