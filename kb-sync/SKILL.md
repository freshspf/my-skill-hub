## 执行步骤

### 0. 前置检查

**先确认每个新模块的 `progress.md` 已填写**（至少有"已掌握知识点"内容），再执行后续步骤。

如果某个模块的 `progress.md` 是空的或只有模板，**跳过该模块**，告知用户先填完再同步。

### 1. 扫描目录，找出新增模块

列出当前目录的所有子目录，排除系统目录（`.obsidian`、`.claude`）。

对比 `CLAUDE.md` 知识库结构章节的代码块，找出**未被记录的新模块**。

### 2. 读新模块的 progress.md

对每个新模块，读其 `progress.md`，提取：
- 模块定位（一句话描述）
- 文档完成情况（已完成哪几篇，格式如"00-07 已完成"）
- 核心技术关键词（用于知识储备现状的摘要）

### 3. 更新 CLAUDE.md

**只更新"知识库结构"章节内的目录树代码块**，在已有条目末尾追加新模块，格式与现有条目保持一致：

```
├── <模块目录名>/        <一句话描述>（已完成 NN-NN）
```

**不要触碰** NEVER、Compact Instructions、技术栈速查等其他章节——这些是协作契约，不随模块增减变化。

### 4. 更新 个人画像.md（三处）

**第4节·当前维护的模块表**：追加一行：

```
| `<模块名>` | <角色描述，含关键规模数字> | `<模块目录名>/` |
```

**第5节·当前工作重心**：在知识库建设那一行末尾，补上新模块名和完成状态。

**第8节·知识储备现状·已掌握**：追加一条，格式：

```
- `<模块名>` <核心能力摘要，括号内列关键技术点>
```

---

**注意**：`基础设施doc/` 是基础设施文档目录，不是微服务模块，更新时单独处理（不加入维护模块表，只更新 CLAUDE.md 目录结构和知识储备现状）。
